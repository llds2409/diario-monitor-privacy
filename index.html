Diário Monitor

Política de Privacidade

Última atualização: 08/04/2026

1. Visão geral

O Diário Monitor foi desenvolvido para ajudar usuários a acompanhar termos cadastrados em publicações do Diário Oficial e receber alertas automáticos quando novas ocorrências forem encontradas.

Esta Política de Privacidade explica quais dados podem ser tratados pelo aplicativo, para que finalidade eles são utilizados e como o usuário pode solicitar esclarecimentos.

2. Dados tratados pelo aplicativo

O Diário Monitor pode tratar os seguintes dados:

  • dados de autenticação informados pelo usuário, como e-mail e senha, por meio da infraestrutura de autenticação do Supabase
  • termos monitorados cadastrados pelo usuário
  • ocorrências encontradas nas fontes monitoradas e associadas aos termos cadastrados
  • preferências de notificação definidas pelo usuário
  • token de dispositivo necessário para envio de notificações push no Android
  • registros técnicos relacionados ao funcionamento do monitoramento automático e sincronização

3. Finalidade do tratamento

Os dados tratados pelo Diário Monitor são utilizados para:

  • autenticar o acesso do usuário
  • salvar e organizar os termos monitorados
  • buscar e registrar ocorrências relacionadas aos termos cadastrados
  • exibir alertas e detalhes das ocorrências encontradas
  • enviar notificações push quando houver novas ocorrências
  • manter o funcionamento técnico, a segurança e a confiabilidade do serviço

4. Fontes de dados monitoradas

Atualmente, o aplicativo utiliza a integração configurada no projeto para consultar resultados relacionados a publicações do Diário Oficial.

As ocorrências exibidas no app podem incluir informações como:

  • título da publicação
  • data
  • seção do DOU
  • página
  • órgão responsável
  • link para a publicação oficial

5. Infraestrutura e servicos de terceiros

O Diário Monitor utiliza servicos de terceiros para operar:

  • Supabase: autenticação, banco de dados e funções de backend
  • Firebase Cloud Messaging (FCM): envio de notificações push no Android

Esses serviços podem tratar dados técnicos estritamente necessários para autenticação, armazenamento e entrega de notificações.

6. Notificações

Quando o usuário ativa notificações no aplicativo, um identificador técnico do dispositivo pode ser armazenado para permitir o envio de alertas.

Esse identificador é utilizado exclusivamente para entrega de notificações relacionadas ao funcionamento do Diário Monitor.

7. Compartilhamento de dados

O Diário Monitor não foi projetado para comercializar dados pessoais dos usuários.

Os dados são utilizados para a operação do aplicativo e podem ser processados apenas na medida necessária pelos serviços de infraestrutura utilizados no projeto.

8. Armazenamento e segurança

Os dados do aplicativo sao armazenados na infraestrutura configurada para o projeto, com uso de mecanismos de autenticação e controle de acesso aplicados no backend.

Embora sejam adotadas medidas técnicas para reduzir riscos, nenhum ambiente conectado à internet pode garantir segurança absoluta.

9. Direitos do usuário

O usuário pode, dentro das funcionalidades disponíveis no aplicativo:

  • revisar termos cadastrados
  • excluir termos monitorados
  • ajustar preferências de notificação

Pedidos adicionais relacionados a dados e suporte podem ser encaminhados para o contato informado pelo responsável do aplicativo.
