Diario Monitor

Politica de Privacidade

Ultima atualizacao: 08/04/2026

1. Visao geral

O Diario Monitor foi desenvolvido para ajudar usuarios a acompanhar termos cadastrados em publicacoes do Diario Oficial e receber alertas automaticos quando novas ocorrencias forem encontradas.

Esta Politica de Privacidade explica quais dados podem ser tratados pelo aplicativo, para que finalidade eles sao utilizados e como o usuario pode solicitar esclarecimentos.

2. Dados tratados pelo aplicativo

O Diario Monitor pode tratar os seguintes dados:

  • dados de autenticacao informados pelo usuario, como e-mail e senha, por meio da infraestrutura de autenticacao do Supabase
  • termos monitorados cadastrados pelo usuario
  • ocorrencias encontradas nas fontes monitoradas e associadas aos termos cadastrados
  • preferencias de notificacao definidas pelo usuario
  • token de dispositivo necessario para envio de notificacoes push no Android
  • registros tecnicos relacionados ao funcionamento do monitoramento automatico e sincronizacao

3. Finalidade do tratamento

Os dados tratados pelo Diario Monitor sao utilizados para:

  • autenticar o acesso do usuario
  • salvar e organizar os termos monitorados
  • buscar e registrar ocorrencias relacionadas aos termos cadastrados
  • exibir alertas e detalhes das ocorrencias encontradas
  • enviar notificacoes push quando houver novas ocorrencias
  • manter o funcionamento tecnico, a seguranca e a confiabilidade do servico

4. Fontes de dados monitoradas

Atualmente, o aplicativo utiliza a integracao configurada no projeto para consultar resultados relacionados a publicacoes do Diario Oficial.

As ocorrencias exibidas no app podem incluir informacoes como:

  • titulo da publicacao
  • data
  • secao do DOU
  • pagina
  • orgao responsavel
  • link para a publicacao oficial

5. Infraestrutura e servicos de terceiros

O Diario Monitor utiliza servicos de terceiros para operar:

  • Supabase: autenticacao, banco de dados e funcoes de backend
  • Firebase Cloud Messaging (FCM): envio de notificacoes push no Android

Esses servicos podem tratar dados tecnicos estritamente necessarios para autenticacao, armazenamento e entrega de notificacoes.

6. Notificacoes

Quando o usuario ativa notificacoes no aplicativo, um identificador tecnico do dispositivo pode ser armazenado para permitir o envio de alertas.

Esse identificador e utilizado exclusivamente para entrega de notificacoes relacionadas ao funcionamento do Diario Monitor.

7. Compartilhamento de dados

O Diario Monitor nao foi projetado para comercializar dados pessoais dos usuarios.

Os dados sao utilizados para a operacao do aplicativo e podem ser processados apenas na medida necessaria pelos servicos de infraestrutura utilizados no projeto.

8. Armazenamento e seguranca

Os dados do aplicativo sao armazenados na infraestrutura configurada para o projeto, com uso de mecanismos de autenticacao e controle de acesso aplicados no backend.

Embora sejam adotadas medidas tecnicas para reduzir riscos, nenhum ambiente conectado a internet pode garantir seguranca absoluta.

9. Direitos do usuario

O usuario pode, dentro das funcionalidades disponiveis no aplicativo:

  • revisar termos cadastrados
  • excluir termos monitorados
  • ajustar preferencias de notificacao

Pedidos adicionais relacionados a dados e suporte podem ser encaminhados para o contato informado pelo responsavel do aplicativo.
